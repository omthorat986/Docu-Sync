========================================
  DOCU-SYNC — REQUIREMENTS & SETUP GUIDE
========================================

What you need installed on your machine before running this project.


────────────────────────────────────────
SYSTEM REQUIREMENTS
────────────────────────────────────────

  Node.js     v18.0.0 or higher   (v20 LTS recommended)
  npm         v8.0.0  or higher   (comes with Node.js)
  Git         any recent version  (to clone the repo)

  MongoDB (choose ONE option):
    Option A — No install needed:
      Leave MONGO_URI blank in your .env file.
      The server auto-starts an in-memory MongoDB (mongodb-memory-server).
      Data is LOST when the server restarts. Good for dev / testing.

    Option B — Local MongoDB:
      Install MongoDB Community Edition 6.0+
      Set MONGO_URI=mongodb://localhost:27017/docusync in backend/.env

    Option C — MongoDB Atlas (cloud, free tier available):
      Create account at https://www.mongodb.com/atlas
      Create a free cluster, get the connection string
      Set MONGO_URI=<your atlas connection string> in backend/.env

────────────────────────────────────────
HOW TO RUN (step by step)
────────────────────────────────────────

1. Clone the repository
   ─────────────────────
   git clone https://github.com/omthorat986/Docu-Sync.git
   cd Docu-Sync

2. Set up backend environment
   ─────────────────────────────
   cd backend
   cp .env.example .env

   Open backend/.env and fill in:

     PORT=5001
     MONGO_URI=           ← leave blank to use in-memory DB (easiest for dev)
     JWT_SECRET=some_random_long_secret_string_here

   Note: JWT_SECRET can be any string. Use something long and random in production.

3. Install backend packages
   ─────────────────────────
   cd backend          (if not already there)
   npm install

4. Start the backend server
   ─────────────────────────
   npm start

   You should see:
     Server running on http://localhost:5001
     MongoDB securely connected!

5. Install frontend packages  (open a NEW terminal)
   ──────────────────────────────────────────────────
   cd frontend
   npm install

6. Start the frontend dev server
   ────────────────────────────────
   npm run dev

   You should see:
     VITE ready in Xms
     Local: http://localhost:5173/

7. Open the app
   ───────────────
   Visit http://localhost:5173 in your browser.
   Sign up for an account and start creating documents!


────────────────────────────────────────
DEFAULT PORTS
────────────────────────────────────────

  Backend  (Express + Socket.IO)   http://localhost:5001
  Frontend (Vite dev server)       http://localhost:5173

  The frontend talks to the backend at http://localhost:5001
  (hardcoded in the source — change if you deploy to different hosts)


────────────────────────────────────────
ENVIRONMENT VARIABLES REFERENCE
────────────────────────────────────────

  File: backend/.env

  Variable         Required   Default    Description
  ─────────────────────────────────────────────────────────────────────
  PORT             No         5001       Port the server listens on
  MONGO_URI        No         (none)     MongoDB connection string.
                                         If blank, uses in-memory DB.
  JWT_SECRET       Yes*       fallback   Secret key for signing JWTs.
                                         *Uses an insecure fallback if missing.
                                          Always set this in production!
